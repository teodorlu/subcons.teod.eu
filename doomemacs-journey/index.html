A Doom Emacs journey

I enjoy the times when I can work in Doom Emacs. I'd like to share that. I tweeted about Emacs, and Doom specifically a while back. Here's the follow-up.

First: Doom Emacs has good documentation, and a community that cares. This page is not the best place to learn about Doom. To learn about Doom, I recommend:

  1. Browsing the official documentation to get an overview, and learn the ropes,
  2. Ask for help in the dedicated #help channel on the Doom Emacs Discord if you get stuck,
  3. Read the guides on the Doom Emacs Discourse to improve fluency with Magit, Dired, or learn to write your Doom package
